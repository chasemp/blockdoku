Blockdoku Placeability Test

Test Instructions

  1. Open the main game in another tab
  2. Place some blocks on the board
  3. Observe that unplaceable blocks are dulled out with an X mark
  4. Place more blocks and verify that placeability updates dynamically
  5. Check that when only one block can be placed, it's highlighted
  6. Verify that when no blocks can be placed, all blocks are dulled out

Expected Behavior

  • ✅ Unplaceable blocks should be grayscale, dimmed, and show an X mark
  • ✅ Placeable blocks should appear normal
  • ✅ When only one block is placeable, it should be highlighted with a pulsing effect
  • ✅ When no blocks are placeable, all blocks should be dulled out
  • ✅ Placeability should update immediately when blocks are placed
  • ✅ Placeability should update when new blocks are generated

CSS Classes Applied

  • .block-item.unplaceable - Applied to blocks that cannot be placed
  • .block-item.placeable - Applied to blocks that can be placed
  • .block-item.last-playable - Applied to the only placeable block

Implementation Details

  • Modified updatePlaceabilityIndicators() to always show placeability state
  • Updated setPlaceability() to support permanent display (durationMs = 0)
  • Enhanced CSS with stronger visual indicators for unplaceable blocks
  • Added placeability updates to generateNewBlocks()
